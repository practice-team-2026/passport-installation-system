# 📋 Паспорт установки

Веб-приложение для учёта смонтированных объектов, паспортов установок и планирования технического обслуживания.

![Python](https://img.shields.io/badge/Python-3.11+-blue.svg)
![Flask](https://img.shields.io/badge/Flask-3.0-green.svg)
![License](https://img.shields.io/badge/License-MIT-yellow.svg)

---

# 📌 О проекте

Система позволяет вести единый реестр смонтированных установок, отслеживать оборудование, планировать техническое обслуживание и анализировать загрузку бригад.

## ✨ Основные возможности

- 📋 Карточки объектов с фотографиями
- 📅 Планирование технического обслуживания
- 📊 Аналитический дашборд с KPI
- 🔍 Поиск и фильтрация объектов
- 🖼️ Загрузка фотографий на сервер
- 🗂️ Ведение паспортов установок

# 🚀 Быстрый старт

## 1. Клонирование репозитория

```bash
git clone git@github.com:practice-team-2026/passport-installation-system.git
cd passport-installation-system
```

---

## 2. Создание виртуального окружения

### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### Linux / macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

---

## 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

---

## 4. Настройка переменных окружения

Создайте файл **.env**

```env
FLASK_APP=run.py
FLASK_ENV=development
SECRET_KEY=your-secret-key
DATABASE_URL=sqlite:///instance/passport.db
```

---

## 5. Создание базы данных

```bash
flask db upgrade
```

---

## 6. Генерация тестовых данных

```bash
python data/seed.py
```

---

## 7. Запуск проекта

```bash
python run.py
```

После запуска приложение будет доступно по адресу:

```
http://localhost:5000
```

# 📊 API

| Метод | Endpoint | Описание |
|--------|----------|----------|
| GET | `/api/installations` | Список объектов |
| GET | `/api/installations/<id>` | Карточка объекта |
| POST | `/api/installations` | Создание объекта |
| PUT | `/api/installations/<id>` | Обновление |
| DELETE | `/api/installations/<id>` | Архивация |
| POST | `/api/upload-photo` | Загрузка фотографии |
| GET | `/api/dashboard/stats` | Данные дашборда |

# 🧪 Тестовые данные

Для демонстрации проекта предусмотрен скрипт генерации тестовых данных.

Он создаёт:

- 30 объектов
- разные статусы (`active`, `draft`, `emergency`, `archived`)
- оборудование (2–4 единицы)
- события технического обслуживания
- адреса и города
- фотографии объектов

Запуск:

```bash
python data/seed.py
```
